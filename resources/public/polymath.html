Planet Math Exporer: A PlayCanvas-based Educational Game for Elementary Math Engagement

As part of my Polymath Jr. REU experience, my team collaborated with Not A Bot, a company that aims to make learning more interactive, and set out to build a math-themed 3D browser game that makes learning engaging for elementary school students (ages 7-10). The game combines interactive gameplay with core math concepts, specifically divisibility, factors, and multiples, wrapped in a playful, outer-space setting.

Built using PlayCanvas, a real-time 3D game engine powered by JavaScript and WebGL, the game allows students to explore planets, make decisions under pressure, and learn by doing within their browser.

Gameplay and Mechanics and User Experience

In the game, players pilot a spaceship using the familiar WASD keyboard controls to navigate a galaxy filled with planets. Each planet is labeled with a number, and the player's ship is also assigned a number. The central mechanic revolves around colliding with planets whose numbers are divisible by the ship's number, encouraging players to use quick mental math and recall divisibility rules. To make the challenge more engaging, the game imposes a fuel-based time limit, which forces players to act accurately and swiftly.

Every player action receives immediate feedback. Correct collisions are rewarded with a cheerful “Yay!” while incorrect ones trigger an “Oh no!” response. Once all valid planets are visited, the player wins the level. This loop of action, feedback, then anticipation supports a satisfying gameplay cycle and keeps players engaged through multiple levels.

Game Architecture and Technical Design

Our game is structured using an Entity-Component System (ECS), which promotes modularity and reuse. Key components include the MovePlayer.js script, which handles user input, ship rotation, and fuel mechanics. The GameLoop.js script manages the overall game state, ensuring smooth transitions across game phases and levels. PlayerGameLogic.js performs the core math checking to validate player interactions, while SetPlanetNumber.js governs each planet’s individual behavior and logic. Finally, UI rendering is managed by FacePlayer.js, which ensures messages like “Try Again” or “You Win!” always appear in the player's field of view.

From a technical standpoint, we built real-time 3D visuals using WebGL via PlayCanvas, implemented physics-based collision detection, and developed a responsive UI system. Our architecture encouraged readable, testable JavaScript code with clear boundaries between gameplay, visual feedback, and logic processing.
