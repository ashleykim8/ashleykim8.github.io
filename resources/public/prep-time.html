PrepTime: Interactive Flashcard System

Prep-Time is a dynamic and intuitive web application I developed with a peer to empower users in creating, managing, and reviewing their own custom flashcard sets. This was also my first experience using React, so it served a dual purpose as a learning opportunity. Designed for efficient learning, it provides a streamlined experience for organizing study materials and fostering knowledge retention.

The application is built with React for a responsive and interactive frontend, leveraging React Router DOM for seamless navigation between different sections. This modular architecture ensures a clean separation of concerns, making the application easy to maintain and scale.

Technical Implementation

I worked with a peer on this project, and I was responsible for the frontend development of PrepTime, focusing on creating a robust and user-friendly flashcard system. Some key technical decisions include:

  • Core Flashcard Management Logic: Implemented the fundamental functionalities allowing users to create new flashcard sets, and within each set, add, edit, and delete individual flashcards. This involved designing the data structures to efficiently store and retrieve flashcard content.
  • State Management with useState: Utilized React's useState hook extensively to manage all dynamic data within the application. This includes tracking the comprehensive list of flashcard sets, input values for new sets and flashcards, and the edit states for individual items. This approach ensures that all changes are immediately reflected across the user interface.
  • Dynamic Routing with react-router-dom: Implemented react-router-dom to enable smooth and dynamic navigation. I used the useNavigate hook for navigating between pages (f.e. from the sets list to a specific flashcard set's detail page) and useParams to extract unique set IDs from the URL, ensuring the correct set's data is displayed.
  • Efficient Data Flow via Props: Designed a clear and efficient data flow using React props. The parent App.js component centrally stores the list of flashcard sets, passing this data and corresponding state update functions (setFlashcardSets) down to child components like FlashcardSets and Flashcards.
  • Component-Based Architecture: Structured the application into two main, distinct components: the FlashcardSets component (for overall set management) and the Flashcards component (for detailed flashcard editing within a set). This division significantly enhances the application's maintainability, readability, and scalability.

This project was an opportunity for me to gain experience with React, state management, and building interactive web applications. It also highlighted the importance of modular component design for creating scalable and maintainable software to me.
